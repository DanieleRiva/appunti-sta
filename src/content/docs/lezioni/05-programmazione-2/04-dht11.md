## Il Sensore DHT11

Il **DHT11** è un sensore digitale che misura contemporaneamente **temperatura** e **umidità relativa** dell'ambiente circostante. È uno dei sensori più diffusi nei progetti Arduino ed è riconoscibile dalla sua caratteristica scocca blu con una griglia frontale.

---

## Collegamento

Il DHT11 ha **4 piedini**, numerati da sinistra a destra guardando il lato frontale (quello con la griglia):

| Piedino |  Nome   | Collegamento                        |
| :-----: | :-----: | :---------------------------------- |
|    1    | **VCC** | **5V** di Arduino                   |
|    2    | **SDA** | Pin digitale di Arduino (es. pin 6) |
|    3    | **NC**  | Non collegato (Not Connected)       |
|    4    | **GND** | **GND** di Arduino                  |

Il piedino 3 (**NC**) si lascia sempre scollegato; non serve a nulla e il suo nome lo dice esplicitamente.

## La Libreria DHT

Per comunicare con il DHT11 si usa la libreria **DHT sensor library** di Adafruit. Va installata prima di poter usare il sensore.

**Ma cos'è esattamente una libreria?**
In programmazione, una libreria è una raccolta di codice già scritto, testato e messo a disposizione dalla community di sviluppatori. Immaginala come un manuale aggiuntivo che insegna ad Arduino comandi che prima non conosceva. Invece di dover **reinventare la ruota** e programmare da zero tutta la logica della "battaglia navale" e dei ritardi di lettura, la libreria fa tutto il lavoro sporco dietro le quinte.

### Installazione

1. Aprire l'**Arduino IDE**
2. Andare su **Sketch → Include Library → Manage Libraries...**
3. Cercare `DHT sensor library` di **Adafruit**
4. Cliccare **Install** e accettare anche l'installazione delle dipendenze se richiesto

### Inclusione nel codice

```cpp
#include "DHT.h"

#define DHTPIN  6       // Pin a cui è collegato il SDA
#define DHTTYPE DHT11   // Modello del sensore

DHT dht(DHTPIN, DHTTYPE); // Crea l'oggetto sensore
```

Queste righe vanno scritte **in cima al file**, prima di `setup()`.

## Lettura dei Dati

Una volta inclusa la libreria e creato l'oggetto `dht`, si usano due funzioni per leggere i valori:

```cpp
float umidita     = dht.readHumidity();    // Umidità in %
float temperatura = dht.readTemperature(); // Temperatura in °C
```

Entrambe restituiscono un `float`. Se la lettura fallisce (sensore scollegato, comunicazione disturbata), restituiscono il valore speciale `NaN` (Not a Number). È buona pratica controllare sempre che la lettura sia valida prima di usare i valori:

```cpp
if (isnan(umidita) || isnan(temperatura)) {
  Serial.println("Errore: impossibile leggere dal sensore!");
  return; // Esce dalla funzione e riprova al prossimo ciclo
}
```

## Il Timing: perché usare `millis()`

Il DHT11 ha una limitazione importante: **non può essere interrogato più velocemente di una volta ogni 2 secondi**. Se si prova a leggerlo più spesso, il sensore restituisce errori o valori NaN.

Per questo motivo non si può semplicemente mettere la lettura dentro `loop()` con un `delay(2000)`: bloccherebbe tutto il programma per due secondi. La soluzione corretta è usare `millis()`:

```cpp
unsigned long previousMillis = 0;
const long interval = 2000; // Lettura ogni 2 secondi

void loop() {
  unsigned long currentMillis = millis();
  if (currentMillis - previousMillis >= interval) {
    previousMillis = currentMillis;
    // Qui si legge il sensore
  }
}
```

## Esempio Completo

```cpp
#include "DHT.h"

#define DHTPIN  6
#define DHTTYPE DHT11

DHT dht(DHTPIN, DHTTYPE);

unsigned long previousMillis = 0;
const long interval = 2000;

void setup() {
  Serial.begin(9600);
  dht.begin();
  Serial.println("Sensore DHT11 pronto.");
}

void loop() {
  unsigned long currentMillis = millis();

  if (currentMillis - previousMillis >= interval) {
    previousMillis = currentMillis;

    float h = dht.readHumidity();
    float t = dht.readTemperature();

    if (isnan(h) || isnan(t)) {
      Serial.println("Errore di lettura!");
      return;
    }

    Serial.print("Umidità: ");
    Serial.print(h);
    Serial.print(" %\t");
    Serial.print("Temperatura: ");
    Serial.print(t);
    Serial.println(" °C");
  }
}
```

:::danger[Intervallo minimo di 2 secondi]
Non abbassare mai `interval` sotto i 2000 ms con il DHT11. Il sensore ha bisogno di tempo per preparare una nuova lettura. Interrogarlo troppo spesso produce errori e non migliora la precisione in nessun modo.
:::
